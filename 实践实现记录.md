# RMDB 实践实现记录

## 题目一：存储管理

- 需求摘要：实现磁盘文件/page 读写、LRU 缓冲池替换、BufferPool 页面生命周期、记录插删改查和表扫描。
- 实现策略：磁盘层用 `lseek/read/write/open/close/unlink` 完成文件操作；缓冲池统一维护 `page_table_`、`free_list_`、LRU 和脏页刷盘；记录层用页头空闲链表加 bitmap 管理 slot。
- 关键文件：`src/storage/disk_manager.cpp`、`src/storage/buffer_pool_manager.cpp`、`src/replacer/lru_replacer.cpp`、`src/record/rm_file_handle.cpp`、`src/record/rm_scan.cpp`。
- 验证结果：`build/bin/unit_test` 通过 5/5；完整构建暂受环境缺少 readline 开发头影响，服务端已移除未使用的 readline include。

## 题目二：查询执行

- 需求摘要：实现基础 DDL/DML/DQL，包括建删表、单表条件查询、更新、删除和连接查询。
- 实现策略：补元数据打开/关闭/drop table；在 Analyze 阶段检查表/列/类型并生成 update set；执行层先以顺序扫描为主，实现条件过滤、投影、嵌套循环连接、delete/update。
- 关键文件：`src/system/sm_manager.cpp`、`src/analyze/analyze.cpp`、`src/execution/executor_*.h`、`src/rmdb.cpp`。
- 验证结果：`cmake --build build -j` 成功；`build/bin/unit_test` 通过 5/5；服务端 socket 冒烟通过 create/insert/select/update/delete/join，非法 SQL 进入异常路径并写 `failure`。

## 题目三、四：BIGINT 与 DATETIME

- 需求摘要：扩展类型系统，支持 8 字节 `BIGINT` 和 `DATETIME`，其中日期时间以 `'YYYY-MM-DD HH:MM:SS'` 输入输出并严格校验。
- 实现策略：`ColType`、`Value`、parser、planner、analyzer、执行器、输出和索引比较统一增加新类型；`DATETIME` 内部编码为 64 位整数 `YYYYMMDDHHMMSS`，输入时校验闰年、日期范围和时间字段。
- 关键文件：`src/defs.h`、`src/common/common.h`、`src/parser/lex.l`、`src/parser/yacc.y`、`src/analyze/analyze.cpp`、`src/execution/execution_manager.cpp`、`src/index/ix_index_handle.h`。
- 验证结果：`cmake --build build -j` 成功；`build/bin/unit_test` 通过 5/5；服务端验证 BIGINT 大值比较、DATETIME 范围比较和非法日期 `2023-02-29 00:00:00` 拒绝。

## 题目五：唯一索引

- 需求摘要：补全 B+ 树基础插查删、create/drop/show index、唯一约束、联合索引和 DML 索引同步。
- 实现策略：实现 B+ 树节点二分查找、叶子点查、插入、分裂、父节点递归插入和边界定位；系统层创建索引时扫描表数据回填并检查重复键；insert/delete/update 同步维护所有索引；show index 按题面格式输出唯一索引元数据。
- 关键文件：`src/index/ix_index_handle.cpp`、`src/index/ix_defs.h`、`src/system/sm_manager.cpp`、`src/parser/yacc.y`、`src/optimizer/optimizer.h`、`src/execution/executor_insert.h`、`src/execution/executor_delete.h`、`src/execution/executor_update.h`、`src/execution/executor_index_scan.h`。
- 验证结果：`cmake --build build -j` 成功；`build/bin/unit_test` 通过 5/5；服务端验证 `create index`、`show index from warehouse`、重复键插入失败、索引列查询、update/delete 后索引同步、联合索引创建和 drop index。

## 题目六：聚合函数

- 需求摘要：支持 `SUM`、`MAX`、`MIN`、`COUNT(col)`、`COUNT(*)` 和 `AS` 别名，无 `GROUP BY`，聚合结果输出单行。
- 实现策略：扩展 select target AST 和 analyzer，识别聚合目标并生成 `AggregatePlan`；执行器遍历子计划所有记录后产出一条聚合记录；普通 select 仍走原 Projection 路径。
- 关键文件：`src/parser/ast.h`、`src/parser/lex.l`、`src/parser/yacc.y`、`src/analyze/analyze.cpp`、`src/optimizer/plan.h`、`src/optimizer/planner.cpp`、`src/execution/executor_aggregate.h`、`src/portal.h`。
- 验证结果：`cmake --build build -j` 成功；服务端验证 `SUM(id)` 输出 `9`、`SUM(val)` 输出六位小数、`MAX/MIN`、`COUNT(*)`、`COUNT(col)` 及带 where 的聚合。

## 题目七：排序与 LIMIT

- 需求摘要：支持多列 `ORDER BY`、每列 `ASC/DESC` 和 `LIMIT` 截断输出。
- 实现策略：AST/parser 将 order by 扩展为多键列表并给 select 增加 limit；`SortPlan/SortExecutor` 收集子计划结果，按多键稳定排序后按 limit 截断，未指定排序但指定 limit 时保持子计划原顺序截断。
- 关键文件：`src/parser/ast.h`、`src/parser/lex.l`、`src/parser/yacc.y`、`src/optimizer/plan.h`、`src/optimizer/planner.cpp`、`src/execution/execution_sort.h`、`src/portal.h`。
- 验证结果：`cmake --build build -j` 成功；`build/bin/unit_test` 通过 5/5；服务端验证 `order by company desc, order_number asc`、`order by order_number asc limit 2` 和无排序 `limit 2`。

## 题目八：事务控制语句

- 需求摘要：支持显式 `begin`、`commit`、`abort/rollback`，隐式单条 SQL 仍自动提交；显式事务内 DML 可提交或回滚，索引与表数据保持一致。
- 实现策略：DML 执行器为 insert/delete/update 追加写集，记录旧记录和每个索引的 old/new key；事务 abort 时逆序反做表记录和索引项，commit/abort 后清理写集；连接循环在隐式语句提交后、显式事务结束后重新创建隐式事务。
- 关键文件：`src/transaction/txn_defs.h`、`src/transaction/transaction_manager.cpp`、`src/execution/executor_insert.h`、`src/execution/executor_delete.h`、`src/execution/executor_update.h`、`src/rmdb.cpp`。
- 验证结果：`cmake --build build -j` 成功；服务端验证无索引 insert/delete 回滚、有索引 insert/update/delete 回滚、commit 保留数据、重复键失败后表和索引仍一致。

## 题目九：基于 no-wait 2PL 的可串行化

- 需求摘要：支持并发事务，使用两阶段封锁保证可串行化；锁冲突时采用 no-wait 立即中止事务，并向客户端返回 `abort\n`。
- 实现策略：实现锁管理器的 S/X/IS/IX 兼容矩阵、锁表、升级和释放；commit/abort 释放事务持有锁；执行器采用保守表级锁策略，select/index scan 加表 S 锁，insert/delete/update 加表 X 锁，范围查询天然由表锁规避幻读。
- 关键文件：`src/transaction/concurrency/lock_manager.h`、`src/transaction/concurrency/lock_manager.cpp`、`src/transaction/transaction_manager.cpp`、`src/execution/executor_seq_scan.h`、`src/execution/executor_index_scan.h`、`src/execution/executor_insert.h`、`src/execution/executor_delete.h`、`src/execution/executor_update.h`。
- 验证结果：`cmake --build build -j` 成功；双客户端验证写锁期间读事务立即收到 `abort`、事务回滚后数据恢复、读范围持有表 S 锁时并发 insert 立即 `abort`、锁释放后 insert/select 正常。

## 题目十：系统故障恢复

- 需求摘要：实现 WAL、redo/undo 日志和启动恢复；crash 后重启时恢复到只包含已提交事务影响的一致状态，并保持索引一致。
- 实现策略：补齐 begin/commit/abort/insert/delete/update 日志记录，DML 记录表名、rid、旧/新记录物理内容；DML 日志立即刷盘，commit/abort 先写日志再刷盘；启动时顺序扫描 `db.log`，redo 已提交事务的 DML，逆序 undo 未完成事务，同步维护表上所有索引。
- 关键文件：`src/recovery/log_manager.h`、`src/recovery/log_manager.cpp`、`src/recovery/log_recovery.h`、`src/recovery/log_recovery.cpp`、`src/transaction/transaction_manager.cpp`、`src/execution/executor_insert.h`、`src/execution/executor_delete.h`、`src/execution/executor_update.h`、`src/record/rm_file_handle.cpp`、`src/record/rm_file_handle.h`、`src/index/ix_index_handle.h`。
- 验证结果：`cmake --build build -j` 成功；`build/bin/unit_test` 通过 5/5；服务端 crash/restart 验证提交 insert 保留、未提交 insert 撤销、提交 update/delete 生效、未提交 update 撤销，重启后索引列查询与全表查询一致。
